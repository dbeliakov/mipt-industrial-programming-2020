Тестирование программного обеспечения

  • Тестирование программного обеспечения — проверка соответствия между реальным и ожидаемым поведением программы, осуществляемая на конечном наборе тестов, выбранном определенным образом
  • Одна из техник контроля качества

Цели тестирования

  • Повысить вероятность того, что ПО, будет работать правильно при любых обстоятельствах
  • Повысить вероятность того, что ПО будет соответствовать всем описанным требованиям
  • Предоставление актуальной информации о состоянии ПО на данный момент

Виды тестирования

  • Автоматизированное - unit-тесты, интеграционные тесты, ...
  • Ручное - самостоятельно или с привлечением QA

Принципы тестирования

  • Тестирование может показать, что дефекты присутствуют, но не может доказать, что их нет.
  • Полное тестирование с использованием всех комбинаций вводов и предусловий физически невыполнимо, за исключением тривиальных случаев
  • Чтобы найти дефекты как можно раньше, активности по тестированию должны быть начаты как можно раньше в жизненном цикле разработки ПО

Принципы тестирования

  • Усилия тестирования должны быть сосредоточены пропорционально ожидаемой, а позже реальной плотности дефектов по модулям
  • Тестовые сценарии должны регулярно рецензироваться и корректироваться, новые тесты должны быть разносторонними, чтобы охватить все компоненты ПО
  • Тестирование выполняется по-разному в зависимости от контекста
  • Обнаружение и исправление дефектов не помогут, если созданная система не подходит пользователю и не удовлетворяет его ожиданиям и потребностям

Статическое тестирование

  • Статическое тестирование производится без запуска исполняемого кода ПО
  • Код-ревью, линтинг исходного кода, анализ скомпилированного кода
  • Позволяет найти в том числе ошибки, тяжело обнаруживаемые при динамическом тестировании
